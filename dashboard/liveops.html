Triage vs. Forecast — Why Two Engines

The feed deliberately does not call the ML models. An incident already on the board already has a known priority flag and a known closure status — those are facts, not predictions, so triage is a simple, direct lookup (8 personnel for a High-priority incident, +2 if a closure is already in effect). The Logistic Regression and Ridge regression models exist for the opposite problem: an event that hasn't happened yet, on the Forecast page, where cause, corridor, and time are known but priority and closure are not. Forecasting and triage are different problems with different available information, so Event Plan Optima treats them as two distinct tools sharing the same underlying historical profile — and the feedback loop above is what ties them together: every resolved incident is a data point on how good the forecast side actually is.
